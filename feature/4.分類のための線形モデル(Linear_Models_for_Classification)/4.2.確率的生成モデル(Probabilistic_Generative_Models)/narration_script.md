## Scene 1: 生成モデルで分類を見る

**画面**

赤と青の 2 クラス散布図に、各クラスの密度を表す楕円を重ねる。右側に `p(x|C_k)`、`p(C_k)`、Bayes、`p(C_k|x)` の流れを縦に並べる。

**ナレーション**

四章二節では、分類を確率モデルとして見直します。
識別関数では、入力エックスを見て、すぐにクラスを分ける面を置きました。
ここではまず、各クラスがどんなデータを生成しやすいか、ピーエックス、バー、シーケーをモデル化します。
さらにクラスの事前確率、ピーシーケーも合わせます。
最後にベイズの定理で、入力エックスが来たときの事後確率ピーシーケー、バー、エックスを計算します。

**原文参照**

PRML p.196-197 の 4.2 冒頭は、分類の確率的見方として、クラス条件付き密度 `p(x|C_k)` と事前確率 `p(C_k)` をモデル化し、ベイズの定理で posterior `p(C_k|x)` を得る生成的アプローチを導入している。

## Scene 2: 二クラスではロジットがシグモイドに入る

**画面**

左に 1 次元の `p(x|C_1)p(C_1)` と `p(x|C_2)p(C_2)` を表示し、評価点 `x` を動かす。右に log odds、sigmoid、posterior の式と sigmoid 曲線を表示する。

**ナレーション**

二クラスの場合、クラス一の事後確率は、尤度と事前確率の積を正規化したものです。
この式は、クラス一とクラス二の比の対数、つまりログオッズを使うと、ロジスティックシグモイドの形に書き換えられます。
エーが大きいほど、クラス一を支持する証拠が強いことを意味します。
エーがゼロなら両クラスは同じだけ支持され、事後確率は一対二、つまりゼロ点五です。
重要なのは、エーが単純な関数になるかどうかです。

**原文参照**

PRML Eq. (4.57)-(4.61) は二クラス posterior を `sigma(a)` として書き、`a` を `p(x|C_1)p(C_1)` と `p(x|C_2)p(C_2)` の log odds として定義する。Fig. 4.9 は logistic sigmoid の形を示す。

## Scene 3: 共有共分散ガウスから線形境界が出る

**画面**

2D 平面に赤青 2 クラスの点、共有共分散を持つガウスの楕円、線形境界を表示する。右側に `p(x|C_k)=N(x|mu_k,Sigma)`、`p(C_1|x)=sigma(w^T x+w_0)`、`w=Sigma^{-1}(mu_1-mu_2)` を出す。

**ナレーション**

連続入力の例として、各クラスの条件付き密度をガウス分布とします。
すべてのクラスが同じ共分散行列シグマを共有していると仮定します。
ガウス分布の指数部には、エックスの二次項が含まれます。
しかし二つのクラスで共分散が同じなら、比を取ると二次項が打ち消されます。
その結果、シグモイドの中身は、ダブリュー転置エックス足すダブリューゼロという線形関数になります。

**原文参照**

PRML Eq. (4.64) は共有共分散ガウスのクラス条件付き密度を置き、Eq. (4.65)-(4.67) で posterior が `sigma(w^T x+w_0)` になることを示す。二次項の相殺により境界が線形になる。

## Scene 4: 事前確率は境界を平行移動する

**画面**

同じ 2D 散布図で、`p(C_1)` スライダーを 0.25、0.50、0.75 に動かし、境界線が平行移動する様子を見せる。

**ナレーション**

事前確率ピーシーケーは、データを見る前に、どのクラスがどれくらい出やすいかを表します。
共有共分散ガウスの場合、事前確率はバイアス項ダブリューゼロにだけ入ります。
クラス一の事前確率を大きくすると、クラス一と判定される領域が広がります。
逆に小さくすると、クラス一にはより強い証拠が必要になります。
重みベクトルの向きは変わらないので、境界は平行に動きます。

**原文参照**

PRML p.199 は、prior `p(C_k)` は `w_0` にだけ入るため、事前確率を変えると posterior の等高線や決定境界が平行移動すると述べている。

## Scene 5: 多クラスではソフトマックスになる

**画面**

三クラスの 2D 点群と線形境界を表示する。右側に softmax の式と、ある点での posterior bar を表示する。

**ナレーション**

クラスが三つ以上ある場合も考え方は同じです。
各クラスについて、ピーエックス、バー、シーケーとピーシーケーの積を作り、全クラスで正規化します。
対数を取ったスコア、エーケーが線形なら、事後確率はソフトマックスで表されます。
共有共分散のガウス分布では、ここでも二次項が打ち消され、エーケーは線形関数になります。
最も大きい事後確率どうしが等しいところが、線形の決定境界です。

**原文参照**

PRML Eq. (4.62)-(4.63) は多クラス posterior を normalized exponential、つまり softmax で表す。Eq. (4.68)-(4.70) は共有共分散ガウスで `a_k(x)` が線形になることを示す。

## Scene 6: 最尤推定で部品を学習する

**画面**

ラベル付きデータをクラスごとに分け、クラス平均、事前確率、共有共分散を式とともに表示する。

**ナレーション**

生成モデルでは、まずクラスごとの密度と事前確率をデータから推定します。
二クラスの共有共分散ガウスなら、クラス一の事前確率パイは、クラス一のデータ数を全データ数で割った値です。
平均ミュー一とミュー二は、それぞれのクラスに属する入力ベクトルの平均です。
共有共分散シグマは、クラスごとの共分散を、データ数で重み付けして平均します。
ただしガウス分布の最尤推定なので、外れ値には強くありません。

**原文参照**

PRML Eq. (4.71)-(4.80) は、二クラス・共有共分散ガウスの尤度と、最尤推定としての prior、クラス平均、共有共分散の解を導く。末尾でガウスの最尤推定は outlier に robust でないことも述べる。

## Scene 7: 離散特徴ではナイーブベイズが線形スコアを作る

**画面**

左に `2^D` 個の状態表を表示し、右にクラスノードから各特徴へ伸びる naive Bayes 構造を表示する。下にベルヌーイ積と log score を表示する。

**ナレーション**

入力がゼロか一の離散特徴でできている場合を考えます。
特徴がディー個あると、一般の表は二のディー乗個の状態を持ち、すぐに大きくなります。
そこで、クラスが決まれば各特徴は条件付き独立だと仮定します。
このナイーブベイズの仮定では、クラス条件付き分布は特徴ごとのベルヌーイ分布の積になります。
対数を取ると、スコアは各エックスアイの線形結合として表されます。

**原文参照**

PRML p.202 は binary feature に対して naive Bayes assumption を置き、Eq. (4.81) の積形式と Eq. (4.82) の線形な `a_k(x)` を得る。

## Scene 8: 生成モデルから識別モデルへの橋渡し

**画面**

Gaussian、Naive Bayes、Exponential family の三つから `a_k(x)=w_k^T x+w_{k0}`、sigmoid / softmax へつながる図を表示する。右側に linear と quadratic の境界例を小さく表示し、次節への橋渡しを入れる。

**ナレーション**

この節の結論は、かなり広い生成モデルから、線形スコアのシグモイドやソフトマックスが自然に出てくる、ということです。
共有共分散のガウス、ナイーブベイズ、そしてスケールを共有した指数型分布族は、その代表例です。
一方で、クラスごとに別の共分散を許すと、二次項は打ち消されず、境界は二次曲線になります。
また、密度の仮定がデータに合わないと、事後確率も悪くなります。
次の確率的識別モデルでは、密度を別々に作らず、事後確率の形を直接学習します。

**原文参照**

PRML Eq. (4.83)-(4.86) は、スケールを共有した指数型分布族でも `a_k(x)` が線形になることを示す。4.3 冒頭では、生成的に密度と prior を別々に推定する代わりに、generalized linear model のパラメータを直接最尤推定する確率的識別モデルへ進む。
