## Scene 1: 識別関数は直接クラスを選ぶ

**画面**

3 クラスの点群に線形境界を重ねる。右側に `x -> y1(x), y2(x), y3(x) -> argmax` の流れを表示し、分類の 3 アプローチのうち「直接: 識別関数」を強調する。

**ナレーション**

第4章は、分類のための線形モデルを扱います。入力ベクトル x を、離散的なクラス C k のどれかへ割り当てる問題です。
その中で4.1節の識別関数は、確率をいったん作らず、入力から直接クラスを選びます。
複数クラスなら、クラスごとにスコア y k x を計算し、一番大きいスコアのクラスを選ぶ、という見方ができます。
これは、事後確率 p C k given x を推定する方法や、生成モデル p x given C k からベイズの定理を使う方法とは違う入口です。
まずは、決定境界が線形になる識別関数を、幾何で理解します。

**原文参照**

PRML p.179-181 は、分類では入力空間が decision regions に分割され、その境界が decision boundaries または decision surfaces と呼ばれることを説明する。4.1 冒頭では、識別関数を「入力ベクトルを K 個のクラスの一つへ割り当てる関数」として導入している。

## Scene 2: 二クラス線形識別関数の幾何

**画面**

二クラス点群と直線境界 `y(x)=0` を表示する。境界に垂直な `w`、境界から点までの符号付き距離、`w0` による平行移動を表示する。

**ナレーション**

二クラスの場合、もっとも単純な線形識別関数は、y x イコール w 転置 x プラス w ゼロです。
y x がゼロ以上ならクラス1、ゼロ未満ならクラス2に割り当てます。
境界は y x イコールゼロなので、二次元では直線、高次元では超平面になります。
重みベクトル w は、この境界に垂直です。つまり w が境界の向きを決めます。
一方、バイアス w ゼロは、境界を原点からどれだけずらすかを決めます。
さらに y x を w の長さで割ると、境界からの符号付き距離になります。

**原文参照**

PRML p.181-182 の Eq. (4.4) は `y(x)=w^T x+w0` を定義する。Eq. (4.5) は原点から境界までの距離、Eq. (4.7) は一般の点から境界までの符号付き距離が `y(x)/||w||` になることを示す。

## Scene 3: 多クラスではスコアを同時に比較する

**画面**

左に二値識別器を組み合わせた曖昧領域を描き、右に `K` 個の線形スコアを同時に比べる多クラス識別器を描く。

**ナレーション**

多クラス分類を、二クラス識別器の寄せ集めで作ることも考えられます。
しかし one versus rest や one versus one を単純に組み合わせると、どのクラスとも言い切れない、あいまいな領域が生じることがあります。
PRML 4.1では、この問題を避けるため、K 個の線形関数 y k x をまとめて使います。
入力 x は、y k x が他のすべての y j x より大きいとき、クラス C k に割り当てられます。
二つのクラスの境界は y k x イコール y j x で表されるので、やはり線形境界です。
この形式では、それぞれの決定領域は凸な領域になります。

**原文参照**

PRML p.182-184 は one-versus-the-rest と one-versus-one の曖昧領域を Fig. 4.2 で示し、Eq. (4.9) の `yk(x)=wk^T x+wk0` による同時比較を導入する。Eq. (4.10) はクラス間境界が線形であることを示す。

## Scene 4: 最小二乗を分類に使うと何が起きるか

**画面**

二クラスデータに最小二乗の境界を引き、正しい側にある追加点で境界が大きく動く様子を示す。右側に閉形式解と「出力が 0 から 1 を外れる」バーを表示する。

**ナレーション**

次に、識別関数のパラメータをどう学習するかを考えます。最初の候補は、回帰で使った最小二乗です。
1 of K の target ベクトルに対して、予測 W 転置 x チルダ が target に近づくように二乗誤差を最小にします。
この方法は、擬似逆行列を使って W イコール X チルダ dagger T という閉形式解を持ちます。
ただし、分類では問題があります。出力の和が1になっても、各成分が0から1の範囲に収まるとは限りません。
さらに二乗誤差は、正しい側に大きく離れた点まで強く罰するため、外れ値に境界が引っぱられます。
閉形式で速いことと、分類に適した損失であることは、別の問題です。

**原文参照**

PRML p.184-186 の Eq. (4.15)-(4.17) は最小二乗の閉形式解を示す。p.185 は 1-of-K coding では出力和の制約は保たれるが、各出力が `[0,1]` に入る保証はないと説明する。Fig. 4.4 と Fig. 4.5 は最小二乗分類の外れ値感度と多クラスでの失敗例を示す。

## Scene 5: Fisher の線形識別

**画面**

二クラス点群を表示し、平均差方向と Fisher 方向への射影を比較する。射影後の点列で重なりの違いを示し、Fisher criterion を表示する。

**ナレーション**

Fisher の線形識別は、二クラスのデータを一つの軸 y イコール w 転置 x に射影する方法として考えます。
単にクラス平均を結ぶ向きへ射影すると、クラス内のばらつきが大きい方向を選んでしまい、射影後に重なりが残ることがあります。
Fisher の考え方は、射影後の平均の差を大きくしつつ、各クラス内のばらつきを小さくすることです。
この比が Fisher criterion で、分子はクラス間の分散、分母はクラス内の分散を表します。
最適な方向は、S W の逆行列に、クラス平均の差を掛けた向きになります。

**原文参照**

PRML p.186-189 の Eq. (4.20)-(4.25) は一次元射影と Fisher criterion を導入する。Eq. (4.26)-(4.30) は `J(w)` を行列で表し、最適方向が `S_W^{-1}(m2-m1)` に比例することを示す。

## Scene 6: Fisher は方向、判定にはしきい値

**画面**

Fisher 軸上に射影した点列へしきい値 `y0` を置く。右側に最小二乗との関係と、多クラス Fisher の最大軸数 `K-1` を表示する。

**ナレーション**

注意したいのは、Fisher が直接クラス名を返す関数ではなく、よい射影方向を与える方法だという点です。
射影後の一次元の値 y に、しきい値 y ゼロを置くことで、はじめて二クラスの判定になります。
PRML ではさらに、二クラスの場合、特別な target coding を使った最小二乗が Fisher と同じ向きを与えることを示します。
また K クラスへ拡張すると、Fisher で得られる有効な線形特徴の数は、最大でも K マイナス1本です。
Fisher は、分類前にクラスを分けやすい低次元表現を作る方法として見ると分かりやすくなります。

**原文参照**

PRML p.189 は Fisher が厳密には識別器ではなく射影方向であり、射影後に threshold を選ぶ必要があると述べる。p.189-190 の Eq. (4.31)-(4.38) は特別な target coding によって最小二乗解が Fisher 方向と一致することを示す。p.191-192 は多クラス Fisher の一般化と、非ゼロ固有値が最大 `K-1` 個であることを説明する。

## Scene 7: パーセプトロン更新

**画面**

二クラス点群と初期境界を表示し、誤分類された点を囲む。その点に対する `w <- w + phi(xn)tn` 更新で境界が動く様子を 2 回示す。

**ナレーション**

最後の例は、Rosenblatt のパーセプトロンです。入力を特徴ベクトル phi x に変換し、w 転置 phi x の符号で二クラスを判定します。
target は、クラス1をプラス1、クラス2をマイナス1とするのが便利です。
正しく分類された点では、重みを変えません。間違えた点だけについて、w に phi x n かける t n を足します。
この更新は、間違えた点を正しい側へ押し戻すように、決定境界を動かします。
データが線形分離可能なら、有限回で正しく分類する解に到達することが保証されます。
ただし、線形分離できないデータでは、パーセプトロンは収束しません。

**原文参照**

PRML p.192-194 の Eq. (4.52)-(4.56) はステップ関数による二クラスモデル、perceptron criterion、確率的勾配降下による更新則を示す。p.194 は線形分離可能な場合の収束定理と、分離不能な場合に収束しないことを説明する。

## Scene 8: 識別関数のまとめ

**画面**

「最小二乗」「Fisher」「Perceptron」の 3 列で、強みと注意点を整理する。最後に次節の確率的線形分類へつなぐ。

**ナレーション**

4.1節の識別関数は、入力からクラスへの直接写像を作る、非確率的な分類方法です。
最小二乗は閉形式で簡単ですが、分類の損失としては外れ値や確率解釈に弱さがあります。
Fisher は、クラス間を離し、クラス内を縮める射影方向を与えます。
パーセプトロンは、誤分類だけを使ってオンラインに重みを更新しますが、線形分離不能な場合の限界があります。
次の節では、p x given C k や p C k given x を扱う、確率的な線形分類へ進みます。

**原文参照**

PRML p.196 は、パーセプトロンが確率的出力を与えず、多クラスへ自然には一般化しづらいこと、固定基底関数の線形結合という制約を持つことを述べている。次節 4.2 は probabilistic generative models へ進む。
