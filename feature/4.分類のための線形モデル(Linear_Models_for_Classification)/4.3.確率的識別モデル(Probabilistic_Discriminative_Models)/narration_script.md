## Scene 1: 事後確率を直接学ぶ

**画面**

左に生成モデルの経路 `p(x|C_k), p(C_k) -> Bayes -> p(C_k|x)`、右に識別モデルの経路 `w -> p(C_k|x)` を並べる。特徴数 `M` に対するパラメータ数の比較をバーで示す。

**ナレーション**

四章の前半では、クラスごとの密度を仮定し、ベイズの定理で事後確率を作りました。
これは生成モデルの考え方です。
一方で分類に必要なのは、最終的には各クラスの事後確率です。
そこで確率的識別モデルでは、条件付き確率ピー、シー、ギブン、エックスを直接モデル化します。
ガウス密度や事前確率を別々に合わせる代わりに、ロジスティックやソフトマックスの形を置き、そのパラメータを直接最尤推定します。
特に特徴数が大きいと、密度全体を作るよりも調整するパラメータが少なくなる利点があります。

**原文参照**

PRML p.203-204 は、4.2 の生成的な間接アプローチに対し、一般化線形モデルの関数形を明示的に使って条件付き分布を直接最尤推定する識別学習を導入している。ロジスティック回帰のパラメータ数が生成的なガウス密度モデルより少なくなる点は p.205 に記載されている。

## Scene 2: 固定基底関数で境界を曲げる

**画面**

左に二次元入力空間の同心円状データを置く。二つのガウス型基底 `phi_1(x)`, `phi_2(x)` へ写す矢印を出し、右の特徴空間では直線で分けられる様子を表示する。

**ナレーション**

ここからは、入力エックスをそのまま使う代わりに、固定された基底関数ファイ、エックスへ変換してから分類します。
モデルは特徴空間では線形です。
けれども、特徴そのものがエックスの非線形変換なので、元の入力空間では曲がった境界になります。
これは回帰の線形基底関数モデルと同じ発想です。
ただし、固定基底はクラスの重なりを魔法のように消すものではありません。
重なりがあるときは、確率そのものをうまく表すことが大事になります。

**原文参照**

PRML p.204-205 の 4.3.1 は、固定非線形基底関数 `phi(x)` を使えば特徴空間では線形、入力空間では非線形な決定境界が得られることを述べる。Fig. 4.12 はその概念図である。

## Scene 3: ロジスティック回帰は分類確率のモデル

**画面**

一次元特徴 `a = w^T phi` 上で、シグモイド曲線 `sigma(a)` を描く。二次元のデータと確率帯を並べ、`p(C_1|phi)=sigma(w^T phi)` と `p(C_2|phi)=1-p(C_1|phi)` を表示する。

**ナレーション**

二クラス分類では、クラス一の事後確率をシグモイドで表します。
線形スコア、エー、イコール、ダブリュー転置ファイを計算し、それをゼロから一の確率へ押し込みます。
シグモイドが二分の一になる場所、つまりエーがゼロの場所が決定境界です。
名前はロジスティック回帰ですが、ここで予測しているのは連続値ではなくクラスの確率です。
その確率を決定理論に渡せば、単にラベルを返すだけでなく、迷いの大きさも扱えます。

**原文参照**

PRML p.205 の Eq. (4.87) は、二クラス分類の事後確率を `p(C1|phi)=sigma(w^T phi)` とするロジスティック回帰を定義している。

## Scene 4: 交差エントロピーと単純な勾配

**画面**

ベルヌーイ尤度から交差エントロピーへ変わる式を表示する。点ごとに `y-t` の符号を矢印で示し、分離可能なデータで `|w|` が大きくなるとシグモイドが急になる様子を見せる。

**ナレーション**

目標値は、クラス一なら一、クラス二ならゼロです。
各点の確率をベルヌーイ分布として掛け合わせ、マイナス対数を取ると、交差エントロピー誤差になります。
この誤差をダブリューで微分すると、とても見通しのよい形になります。
各点の寄与は、予測ワイから目標ティーを引いた誤差に、特徴ベクトルファイを掛けたものです。
ただし、データが完全に線形分離できると、最尤推定では重みの大きさが無限に伸び、シグモイドが階段関数のように急になります。
この特異性は、事前分布や正則化で抑える必要があります。

**原文参照**

PRML p.206 の Eq. (4.89)-(4.91) は、ベルヌーイ尤度、交差エントロピー誤差、勾配 `sum_n (y_n-t_n) phi_n` を与える。p.206-207 は線形分離可能なデータで最尤推定の重みが発散する問題を述べる。

## Scene 5: IRLS は重み付き最小二乗を繰り返す

**画面**

Newton-Raphson の局所二次近似を表示し、ロジスティック回帰の Hessian `Phi^T R Phi` と `R_nn=y_n(1-y_n)` を示す。境界付近の点を大きく、確信度の高い点を小さく描く。

**ナレーション**

線形回帰なら、二乗誤差は二次関数なので、一回の最小二乗で解けました。
ロジスティック回帰ではシグモイドが入るため、閉じた形の解はありません。
そこでニュートン法を使い、現在のダブリューの近くで誤差を二次関数として近似します。
ヘッセ行列には、アール、イコール、ワイかける一マイナスワイ、という重みが出てきます。
この重みは、確率が半々に近い境界付近で大きく、ほぼゼロか一の点では小さくなります。
つまりアイアールエルエスは、不確かな点を重く見ながら、重み付き最小二乗を繰り返す方法です。

**原文参照**

PRML p.207-208 の Eq. (4.92)-(4.100) は、Newton-Raphson 更新、Hessian `Phi^T R Phi`、IRLS 形式の更新を導く。Eq. (4.101)-(4.103) は `R_nn=y_n(1-y_n)` と有効目標 `z_n` の解釈を与える。

## Scene 6: 多クラスでは softmax を使う

**画面**

三クラス点群に三本の線形スコア `a_k=w_k^T phi` を表示し、softmax で確率バーが正規化される様子を示す。1-of-K ターゲットと多クラス交差エントロピーを並べる。

**ナレーション**

クラスが三つ以上あるときは、シグモイドの代わりにソフトマックスを使います。
各クラスに線形スコア、エー、ケーを作り、その指数を全クラスの指数の和で割ります。
こうすると、すべての確率はゼロから一の間に入り、合計は一になります。
目標は、正しいクラスだけが一で、ほかはゼロのワンオブケー符号です。
負の対数尤度は多クラス交差エントロピーになり、勾配はここでも、予測マイナス目標、かける特徴、という同じ形になります。

**原文参照**

PRML p.209-210 の Eq. (4.104)-(4.109) は、softmax、1-of-K 符号化、多クラス交差エントロピー、勾配 `sum_n (y_nj-t_nj) phi_n` を定義する。Eq. (4.110) は多クラス IRLS の Hessian ブロックを示す。

## Scene 7: probit はしきい値ノイズの CDF

**画面**

しきい値 `theta` の分布を青い密度で描き、その累積分布を赤い S 字曲線として示す。logistic と probit を重ね、外れ値が境界の反対側にあるときの感度差を表示する。

**ナレーション**

シグモイド以外の活性化関数も考えられます。
ひとつの見方は、線形スコアがランダムなしきい値シータを超えたらクラス一、という noisy threshold モデルです。
しきい値に分布を置くと、クラス一になる確率は、その分布の累積分布関数になります。
しきい値ノイズを標準ガウスにすれば、活性化関数はプロビット関数です。
多くの場合、ロジスティック回帰とプロビット回帰の結果は似ています。
ただし尾の減り方が違うため、間違った側に遠く離れた外れ値への感度は変わります。
ラベル誤りを明示的に入れるなら、確率イプシロンで目標が反転するモデルとして扱えます。

**原文参照**

PRML p.210-212 の 4.3.5 は、noisy threshold による活性化関数、probit function、外れ値への感度、ラベル反転確率 `epsilon` を含むモデルを説明する。

## Scene 8: canonical link が勾配をそろえる

**画面**

線形回帰、ロジスティック回帰、多クラス softmax を表にし、すべての勾配が `error x feature` の形になることを強調する。最後に 4.3 のまとめを表示する。

**ナレーション**

最後に、なぜ同じ勾配の形が何度も出てきたのかを整理します。
線形回帰では、予測と目標の差に特徴を掛ける形でした。
ロジスティック回帰でも、ソフトマックスでも、対応する交差エントロピーを使うと同じ形になります。
PRML はこれを、指数型分布族と canonical link の組み合わせとして説明します。
平均を線形スコアへ戻すリンク関数を、分布に対応する自然な形に選ぶと、余分な微分項が打ち消されます。
確率を直接学ぶ識別モデルは、この単純な勾配と、事後確率を返せる実用性の両方を持っています。

**原文参照**

PRML p.212-213 の 4.3.6 は、ターゲット変数に指数型分布族を仮定し、canonical link を選ぶことで勾配が `sum_n (y_n-t_n) phi_n` 型へ単純化する一般結果を示す。
