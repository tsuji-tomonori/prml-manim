## Scene 1: オープニング

**画面**

無数の青い点をランダムに出し、ゆっくり波形へ整列させる。タイトル「PRML Chapter 1: データの点から“見えない関数”を探す」を表示する。

**ナレーション**

データは、ただの数字の集まりに見えます。
でも、その中に何度も現れる形があるとき、私たちはそれをパターンと呼びます。
機械学習の最初の仕事は、この形を見つけることです。
ただし、見つけたいのは、過去のデータそのものではありません。
まだ見ていないデータにも通用する、背後の規則性です。

**原文参照**

PRML p.1 は、パターン認識を「データ中の規則性をアルゴリズムで発見し、分類などの行動に使う分野」として導入している。

## Scene 2: 手書き数字から数値ベクトルへ

**画面**

手書きの「3」に見えるピクセルグリッドを表示する。格子を重ね、各ピクセルの濃淡が数値に変わり、右側の縦ベクトルへほどける。`28\times 28 = 784` を表示する。

**ナレーション**

人間には、これは数字の「3」に見えます。
でもコンピュータに渡すとき、それは明るさの数字がたくさん並んでいるだけです。
28 かける 28。
つまり画像1枚は、784個の数字のリストだと思えば十分です。
その数字の並びから、「これは3っぽい」と判断する。
これがパターン認識のひとつの例です。

**原文参照**

PRML p.1-2 と Fig. 1.1 は、手書き数字認識を例にし、28×28 ピクセル画像を 784 個の実数からなる入力ベクトルとして扱うと説明している。

## Scene 3: 学習とは generalization

**画面**

`training set` の箱から `model y(x)` に矢印を出し、さらに `test input` と `prediction` に流す。

**ナレーション**

機械学習で本当にやりたいことは、過去のデータを覚えることではありません。
練習問題を使って、初見問題に強くなることです。
PRML では、この能力を generalization、つまり汎化と呼びます。
訓練データに合うだけでは足りません。
まだ見ていない入力に対して、うまく予測できることが目的です。

**原文参照**

PRML p.2 は、training set でモデルを調整し、test set の新しい数字画像を正しく分類する能力を generalization と説明している。

## Scene 4: 今回は regression

**画面**

classification / regression / unsupervised learning の3つを小パネルで示す。中央の regression を赤く強調し、`x \to t` を表示する。

**ナレーション**

学習問題には、いくつかの種類があります。
画像が 0 から 9 のどれかを当てるなら、分類問題です。
一方で、入力 x から、連続的な数字 t を予測するなら、回帰問題です。
今回の多項式曲線フィッティングは、この回帰問題の小さな実験です。

**原文参照**

PRML p.3 は supervised learning の中で、離散カテゴリを割り当てる問題を classification、連続値を出力する問題を regression と説明している。

## Scene 5: 10個の点から見えない曲線を想像する

**画面**

x-t 平面に青い観測点を10個表示する。しばらく点だけを見せたあと、比較用の緑の `sin(2πx)` を薄く表示する。

**ナレーション**

ここに10個の点があります。
横軸は入力 x、縦軸は予測したい値 t。
この点たちは、完全にバラバラではなさそうです。
何か、波のような形に沿っている気がします。
でも、学習アルゴリズムに見えているのは青い点だけです。
実は今回は、裏側にこの緑の曲線がある、という設定でデータを作っています。
ただし、学習する側にはこの曲線は見えていません。
問題は、有限個の点から、まだ見ていない場所を予測できるかです。

**原文参照**

PRML p.4-5 と Fig. 1.2 は、`sin(2πx)` に Gaussian noise を加えた N=10 の訓練データを使い、新しい x に対する t を予測する問題として 1.1 節を導入している。

## Scene 6: 多項式モデル

**画面**

`y=a`、`y=a+bx`、`y=a+bx+cx^2`、`y=a+bx+cx^2+dx^3+\cdots` の順に表示し、右側で基底曲線が足し合わされて赤い曲線になる。最後に `y(x,\mathbf{w})=\sum_{j=0}^M w_jx^j` を小さく表示する。

**ナレーション**

今回、予想曲線として使うのは多項式です。
まずは水平な線。
次に直線。
次に二次式、三次式。
多項式とは、1、x、xの二乗、xの三乗、という材料を足し合わせて作る曲線です。
どの材料をどこまで使うかを決める数字が、次数 M です。
M が小さいと曲線は硬い。
M が大きいと曲線はよく曲がる。
つまり M は、曲線の自由さを決めるつまみです。

**原文参照**

PRML p.5 の Eq. (1.1) は、多項式モデルを `y(x,w)=w_0+w_1x+\cdots+w_Mx^M` と定義し、M を多項式の次数として導入している。

## Scene 7: 最小二乗

**画面**

3次多項式を赤で表示し、各点から曲線まで黄色の縦線を引く。縦線の一部を正方形に変換し、「外し具合 = ズレ₁² + ズレ₂² + ...」を表示する。最後に PRML の誤差関数を表示する。

**ナレーション**

曲線の良さを測るには、まず点と曲線のズレを見ます。
ズレが小さいほど、曲線は点に合っていると言えます。
ただ足すだけだと、上に外したズレと下に外したズレが打ち消し合ってしまいます。
そこで、ズレを二乗します。
二乗すると、上に外しても下に外してもプラスになります。
しかも、大きく外した点ほど強く罰せられます。
ズレを二乗して、全部足す。
これが最小二乗の考え方です。
画面のシグマは、全部足すという意味の省略記号です。

**原文参照**

PRML p.5-6 の Eq. (1.2) は、各データ点での予測 `y(x_n,w)` と目標値 `t_n` の差の二乗和を誤差関数として定義している。Fig. 1.3 は、そのズレを縦方向の変位として図示している。

## Scene 8: M=0,1,3,9 の比較

**画面**

4分割画面で M=0、M=1、M=3、M=9 の赤い曲線を並べる。M=3 を緑寄りに評価し、M=9 では「点には合うが暴れる」と表示する。

**ナレーション**

では、M を変えると何が起きるでしょうか。
M=0 は水平線しか作れません。これはさすがに硬すぎます。
M=1 は直線です。少し傾けられますが、波の形には追いつけません。
M=3 になると、ちょうどよく曲がれるようになります。
点にもそこそこ合っていて、全体の流れもつかんでいます。
そして M=9。
これは一見すごいです。ほとんど全部の点を通っています。
でも、よく見ると曲線が不自然に暴れています。
点には合っているのに、背後の流れからは外れている。
これが過学習です。
過学習とは、練習問題に合わせすぎて、初見問題に弱くなることです。

**原文参照**

PRML p.6-7 と Fig. 1.4 は、M=0、1、3、9 の比較を示している。本文では M=9 が訓練点を通る一方で激しく振動し、`sin(2πx)` の悪い表現になることを over-fitting と説明している。

## Scene 9: 訓練誤差とテスト誤差

**画面**

横軸 M、縦軸 RMS error のグラフを描く。training を青、test をオレンジで表示する。`E_RMS` の式を補助的に出す。

**ナレーション**

練習問題だけを見ると、M を大きくするほど外し具合は小さくなります。
これは当然です。曲線が自由になるからです。
でも、初見問題では違います。
テスト誤差は途中まで下がったあと、また大きくなってしまいます。
つまり、練習問題で満点を取ることと、初見問題に強いことは同じではありません。
PRML ではこの比較のために RMS error を使います。
これは外し具合を、t と同じくらいのスケールで見やすくしたものです。

**原文参照**

PRML p.6-8 の Eq. (1.3) と Fig. 1.5 は、training set と independent test set の RMS error を M ごとに比較している。本文では M=9 で training error が 0 になる一方、test error が大きくなると説明している。

## Scene 10: 係数の暴走

**画面**

M=3 の係数バーを表示してから、M=9 の係数バーへ変形する。バーが大きく正負に振れる。

**ナレーション**

9次式が点を全部通れるのは、曲線が本当に賢くなったからではありません。
係数をものすごく大きくしたり、小さくしたりして、点の近くで無理やり曲げているからです。
これは、答えの規則を理解したというより、点の位置を丸暗記している状態に近いです。
係数が大きいこと自体が、いつも悪いわけではありません。
ただ、この例では、係数の大きさが曲線の不自然な暴れ方とつながっています。

**原文参照**

PRML p.8-9 と Table 1.1 は、多項式の次数が大きくなると係数の典型的な大きさが増し、M=9 では大きな正負の係数で訓練データに細かく合わせていると説明している。

## Scene 11: データ数を増やす

**画面**

左に N=15、右に N=100 の M=9 フィットを並べる。右側の曲線が安定する様子を見せる。

**ナレーション**

同じ M=9 でも、データの数を増やすと様子が変わります。
点が少ないと、曲線は少数の点に振り回されます。
点が多いと、曲線は全体の流れをつかみやすくなります。
複雑なモデルを使うには、それを支えるデータも必要です。

**原文参照**

PRML p.9 と Fig. 1.6 は、M=9 のまま N=15 と N=100 を比較し、データセットが大きくなるほど over-fitting が弱まることを示している。

## Scene 12: 正則化

**画面**

M=9 の赤い曲線と λ スライダーを表示する。λ を動かすと曲線が滑らかになり、さらに大きくすると硬くなりすぎる。正則化付き誤差関数を表示する。

**ナレーション**

ここまでは、点に合うことだけを考えていました。
でも、それだけだと曲線が暴れすぎることがあります。
そこで、係数を大きくしすぎることにも罰を与えます。
点からのズレを小さくしたい。
でも、係数を大きくしすぎるのも避けたい。
この二つの要求のバランスを取るつまみが λ です。
λ が小さすぎると、点を追いかけすぎて暴れます。
ちょうどよいと、全体の流れをつかみます。
大きすぎると、今度は曲線が硬くなりすぎます。
ここでも大事なのは、ちょうどよい複雑さです。

**原文参照**

PRML p.10-11 の Eq. (1.4) は、二乗誤差に係数の二乗和ペナルティを加えた正則化付き誤差関数を導入している。Fig. 1.7 は λ による M=9 曲線の変化を示し、Fig. 1.8 は λ と RMS error の関係を示している。

## Scene 13: 確率論への橋渡し

**画面**

点の周りにノイズ雲を出し、曲線の周りに不確実性の帯を出す。

**ナレーション**

ここまでは、一本の曲線を選ぶ話でした。
でも実際には、データにはノイズがあります。
つまり、予測には「どれくらい不確かか」もあるはずです。
一本の線だけでなく、不確かさも同時に表したい。
そのために、次に必要になるのが確率論です。

**原文参照**

PRML p.5 は、有限データとノイズのために新しい入力への予測には不確実性があると述べ、Section 1.2 の probability theory へつなげている。p.12 の 1.2 冒頭では、測定ノイズと有限データセットが uncertainty の原因だと説明している。

## 収録用の短縮版ナレーション指針

- 式は画面に任せ、声では意味を読む。
- `\sum` は「全部足す」、`\lambda` は「ブレーキの強さ」と言い換える。
- `training set` は「練習問題」、`test set` は「初見問題」と言い換える。
- `over-fitting` は「練習問題の丸暗記」として説明する。
- `regularization` は「係数の大きさにもコストを払わせる」と説明する。
