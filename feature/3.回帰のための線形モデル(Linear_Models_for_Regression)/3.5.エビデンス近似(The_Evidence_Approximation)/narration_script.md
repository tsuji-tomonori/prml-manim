## Scene 1: エビデンス近似の位置づけ

**画面**

左に完全ベイズの積分 `integrate w, alpha, beta`、右に evidence approximation の `integrate w` と `maximize alpha, beta` を対比する。下部に `alpha / beta = regularization strength` を表示する。

**ナレーション**

ベイズ線形回帰では、重み w を分布として扱いました。
本当は、正則化の強さ alpha や、ノイズ精度 beta にも分布を置き、全部を積分したいところです。
しかし、重みとハイパーパラメータをすべて積分するのは、一般には扱いにくくなります。
そこで、重みは積分し、alpha と beta は、データのエビデンスが最大になる値で代表させます。
この近似を、エビデンス近似、または経験ベイズと呼びます。

## Scene 2: エビデンスとは

**画面**

1 次元の重み軸に likelihood と prior を重ね、積 `p(t|w,beta)p(w|alpha)` の面積を塗る。山の高さだけでなく、幅も evidence に効くことを示す。

**ナレーション**

エビデンスとは、何を測っているのでしょうか。
式では、尤度と事前分布を掛けて、重み w で積分した量です。
よく当てはまる重みがあるだけでは足りません。
事前分布の中で、どれくらい広い重みの領域がデータを説明できるかも効きます。
つまりエビデンスは、当てはまりの良さと、複雑さの使いすぎを同時に見る尺度です。

## Scene 3: log evidence の分解

**画面**

`ln p(t|alpha,beta)` を 5 つの項に分ける。`-E(m_N)` をデータ誤差と重みペナルティへ展開し、`-1/2 ln |A|` を複雑さの体積補正として見せる。

**ナレーション**

線形ガウスモデルでは、重みに関する積分を解析的に計算できます。
対数エビデンスは、alpha と beta の正規化項、データ誤差、重みの大きさ、そしてヘッセ行列の行列式に分かれます。
データ誤差を小さくするほど良い一方で、重みを細かく調整しすぎると、行列式の項が複雑さとして効きます。
このつり合いが、検証データなしで正則化の強さを選ぶ仕組みです。

## Scene 4: alpha を evidence で選ぶ

**画面**

左に回帰曲線、右に `ln p(t|alpha,beta)` 対 `ln alpha` の曲線。alpha スライダーを動かし、曲線が硬い状態、自由すぎる状態、中間で evidence 最大の状態を切り替える。

**ナレーション**

alpha は、重みをゼロの近くに保つ強さです。
alpha が大きすぎると、曲線は硬くなり、データを追えません。
小さすぎると、重みが自由になりすぎ、細かな揺れまで拾います。
エビデンスを alpha の関数として見ると、中間の値で山ができます。
この山の頂点が、訓練データから選ばれる alpha です。

## Scene 5: 有効パラメータ数 gamma

**画面**

固有方向を横に並べ、`lambda_i / (alpha + lambda_i)` のメーターを表示する。大きい曲率は 1 に近く、小さい曲率は 0 に近い。合計を `gamma` として数える。

**ナレーション**

エビデンス近似では、gamma という量が現れます。
各方向について、データが作る曲率 lambda_i が alpha より大きければ、その方向の重みはデータでよく決まります。
逆に、曲率が小さい方向は、事前分布に押し戻され、ほとんど使われません。
gamma は、ゼロから M までの間で動く、有効なパラメータ数です。

## Scene 6: beta の更新

**画面**

残差二乗和のバーを作り、分母が `N` から `N - gamma` に変わる様子を見せる。`1/beta = RSS / (N - gamma)` を大きく表示する。

**ナレーション**

beta は、観測ノイズの精度です。逆数はノイズ分散に対応します。
最尤推定なら、残差二乗和をデータ数 N で割って分散を見積もります。
ベイズ的な更新では、分母が N ではなく、N マイナス gamma になります。
データで決めた有効パラメータ数の分だけ、自由度を差し引くと考えられます。

## Scene 7: 再推定の反復

**画面**

`alpha, beta -> m_N -> gamma -> alpha_new, beta_new` の循環図と、反復ごとの点が evidence 曲線を上る小さなプロットを表示する。

**ナレーション**

実際には、alpha も beta も、gamma や事後平均 m_N に依存します。
そこで、まず初期値を置き、重みの事後平均を計算します。
そこから gamma を求め、alpha と beta を更新します。
この手順を繰り返すと、エビデンスの高い値へ近づいていきます。
交差検証の候補探索ではなく、訓練データの周辺尤度を直接押し上げる流れです。

## Scene 8: まとめ

**画面**

`integrate w`、`maximize evidence`、`gamma = effective parameters`、`estimate alpha and beta from training data` の 4 項目を整理し、次節 `3.6 Limitations of Fixed Basis Functions` へつなぐ。

**ナレーション**

エビデンス近似は、完全ベイズよりは近似的ですが、重みを一点に固定しません。
重みを積分したうえで、ハイパーパラメータを周辺尤度で選びます。
その結果、alpha と beta、つまり正則化とノイズを、訓練データから同時に推定できます。
中心にある直感は、よく当てはまり、しかも必要以上に自由すぎない説明を選ぶ、ということです。
次は、固定された基底関数を使うこと自体の限界へ進みます。
