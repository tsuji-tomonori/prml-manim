## Scene 1: 二値から多値へ

**画面**

Binary パネルから Multinomial パネルへ矢印を出し、手書き数字や文書分類の例を添える。

**ナレーション**

前の節では、表か裏、ゼロか一、という二値変数を見ました。
でも実際の分類では、候補が二つだけとは限りません。
手書き数字なら、ゼロから九までの十通り。
文書分類なら、ニュース、スポーツ、経済、天気、という複数の状態があります。
PRML 2.2 節では、このような多項変数を扱います。

**原文参照**

PRML p.74 は、binary variables の次に、相互排他的な K 個の状態を取る離散変数を導入している。

## Scene 2: 1-of-K 表現

**画面**

6 個の状態カードのうち 3 番目を強調し、`(0,0,1,0,0,0)^T` へ変換する。最後に `sum_k x_k = 1` を表示する。

**ナレーション**

多項変数は、K 個の状態のうち、ちょうど一つだけが選ばれる変数です。
PRML では、これを 1-of-K 表現で書きます。
たとえば 6 通りのうち 3 番目が出たなら、ベクトルは、0, 0, 1, 0, 0, 0 です。
1 が立っている場所だけが、観測された状態を表します。
全ての成分を足すと、必ず 1 になります。

**原文参照**

PRML p.74-75 と Eq. (2.25) は、K=6 で 3 番目の状態を `(0,0,1,0,0,0)^T` と表す例を示している。

## Scene 3: カテゴリ分布

**画面**

確率 `mu_k` の棒グラフを出し、3 番目の棒を選択する。右側に `p(x|mu)=prod_k mu_k^{x_k}` を表示する。

**ナレーション**

それぞれの状態には、起こりやすさを表す `mu_k` を置きます。
`mu` は確率なので、全て 0 以上で、合計は 1 です。
1-of-K 表現を使うと、一回の観測の確率を、積の形で短く書けます。
観測された場所だけ指数が 1 になり、ほかは 0 になります。
つまり式は長く見えても、実際には選ばれた `mu` を一つ取り出しているだけです。

**原文参照**

PRML p.75 の Eq. (2.26) は、1 回の観測に対する分布を `prod_k mu_k^{x_k}` と表している。

## Scene 4: データはカウントに縮約される

**画面**

観測列の丸を並べ、各カテゴリのカウント棒グラフへ集約する。`p(D|mu)=prod_k mu_k^{m_k}` を表示する。

**ナレーション**

独立な観測が何回もあると、尤度は各観測の確率を掛け合わせたものになります。
しかし、全部の順番を覚えておく必要はありません。
この分布では、各状態が何回出たか、というカウントだけで尤度が決まります。
このカウントを `m_k` と書きます。
`m_k` は、この分布にとって十分統計量です。

**原文参照**

PRML p.75 の Eq. (2.29)-(2.30) は、データ集合の尤度がカウント `m_k=sum_n x_{nk}` だけに依存することを示している。

## Scene 5: 最尤推定は相対頻度

**画面**

カウント棒グラフを確率棒グラフへ変換し、`mu_k^ML=m_k/N` を表示する。

**ナレーション**

では、`mu` をデータから推定するとどうなるでしょうか。
`mu` の合計が 1、という制約の下で尤度を最大にします。
結果はとても直感的です。
各 `mu` の最尤推定値は、その状態が出た回数を、全体の回数で割ったものになります。
つまり最尤推定では、確率を観測された相対頻度として読むことになります。

**原文参照**

PRML p.75-76 の Eq. (2.31)-(2.33) は、制約付き最適化により `mu_k^ML=m_k/N` を導出している。

## Scene 6: 多項分布

**画面**

同じカウントを持つ二つの観測列を示し、多項分布の式と係数 `N!/(m_1!...m_K!)` を表示する。

**ナレーション**

今度は、順番ではなく、カウントの組そのものの確率を考えます。
同じカウントでも、観測の並び方はたくさんあります。
その並び方の数を掛けたものが、多項分布です。
係数は、N 個の観測を、K 個のグループへ分ける方法の数を表します。
二値の二項分布を、K 通りに広げたものだと見れば十分です。

**原文参照**

PRML p.76 の Eq. (2.34)-(2.36) は、多項分布と正規化係数、カウント制約 `sum_k m_k=N` を与えている。

## Scene 7: ディリクレ分布

**画面**

`mu_1+mu_2+mu_3=1` の三角単体上に点群を表示し、`Dir(mu|alpha) proportional prod_k mu_k^{alpha_k-1}` を示す。

**ナレーション**

ベイズ的に考えるなら、`mu` そのものにも事前分布を置きます。
`mu` は合計が 1 なので、自由に動ける場所は単体と呼ばれる領域の上だけです。
多項分布に対する便利な事前分布が、ディリクレ分布です。
パラメータの `alpha_k` は、その状態をどれくらい先に信じているかを表します。
`alpha` は、仮想的な事前カウントとして解釈できます。

**原文参照**

PRML p.76-77 の Eq. (2.37)-(2.39) は、Dirichlet 分布を定義し、Fig. 2.4 は K=3 の単体制約を図示している。

## Scene 8: 事前カウントに実データを足す

**画面**

`alpha=(2,1,2)` と `m=(2,2,5)` を足し、`alpha+m=(4,3,7)` の posterior へ更新する。

**ナレーション**

ディリクレ事前分布に多項分布のデータを掛けると、事後分布もディリクレ分布になります。
変わるのはパラメータだけです。
事前の `alpha` に、観測されたカウント `m` を足します。
これは、前の節のベータ分布とベルヌーイ分布で見た関係の、多値版です。
次の節では、連続値を扱う中心的な分布、ガウス分布へ進みます。

**原文参照**

PRML p.77 の Eq. (2.40)-(2.41) は、posterior が `Dir(mu|alpha+m)` になることを示し、Dirichlet が multinomial の共役事前分布であることを説明している。
