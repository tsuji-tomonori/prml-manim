## Scene 1: 逆伝播は学習法ではなく微分の計算手順

**画面**

入力、隠れ層、出力層を並べた小さなネットワークを表示する。青い矢印で forward pass、赤い矢印で backward pass を示す。左下に「gradient evaluation」、右下に「optimizer uses gradients」を分けて表示する。

**ナレーション**

この節の目的は、フィードフォワードネットワークの誤差関数を、すべての重みで効率よく微分することです。
バックプロパゲーションという言葉は、学習アルゴリズム全体を指して使われることもあります。
ここでは、重みをどう更新するかではなく、勾配をどう計算するかに限定して考えます。
入力から出力へ値を流し、そのあと誤差の情報を出力から入力側へ戻します。
前向きと後ろ向きの局所的なメッセージパッシングが、誤差逆伝播です。

**原文参照**

PRML p.241-242 は、backpropagation を gradient descent そのものではなく、誤差関数の微分を効率的に評価する方法として用語整理している。

## Scene 2: 重みの微分は局所情報の積になる

**画面**

1 本の接続 `z_i -> w_ji -> a_j` を拡大し、`a_j = sum_i w_ji z_i`、`delta_j = dE_n/da_j`、`dE_n/dw_ji = delta_j z_i` を順に表示する。

**ナレーション**

まず一つの重みに注目します。重み `w_ji` は、入力側の値 `z_i` を受け取り、ユニット `j` の総入力 `a_j` にだけ直接影響します。
したがって、連鎖律を使うと、誤差の重み微分は、誤差の `a_j` 微分と、`a_j` の重み微分の積になります。
`delta_j` を、誤差を `a_j` で微分した量、と定義します。
また、`a_j` を重み `w_ji` で微分すると、入力側の値 `z_i` が残ります。
つまり、重みの勾配は、出力側のデルタと入力側の値の積、`delta_j z_i` です。

**原文参照**

PRML p.242-243 の Eq. (5.48)-(5.53) は、連鎖律から `partial E_n / partial w_ji = delta_j z_i` を導く。

## Scene 3: 前向き計算で必要な値を保存する

**画面**

同じネットワークで、入力から隠れ層、出力層へ青い矢印を流す。各ユニットの横に `a_j` と `z_j = h(a_j)` の小ラベルを出す。

**ナレーション**

勾配を計算する前に、まず通常の前向き計算を行います。
各ユニットでは、前の層から来た値を重み付きで足し合わせて、総入力 `a` を作ります。
次に、微分可能な非線形関数 `h` を通して、出力 `z` を作ります。
後ろ向き計算では、この `a` と `z` が必要になります。
前向き計算は、予測値を出すだけでなく、微分に使う中間値を準備する段階でもあります。

**原文参照**

PRML p.243 は、各ユニットで `a_j = sum_i w_ji z_i`、`z_j = h(a_j)` を successive application で計算する forward propagation を説明している。

## Scene 4: 出力ユニットのデルタから始める

**画面**

出力ユニットの横に予測 `y_k` と目標 `t_k` を置き、差分から赤い `delta_k` を作る。`delta_k = y_k - t_k` を大きく表示する。

**ナレーション**

後ろ向き計算は出力ユニットから始まります。
二乗和誤差と線形出力、または正準リンクと対応する誤差関数を使う場合、出力のデルタは単純になります。
`delta_k` は、予測値 `y_k` から目標値 `t_k` を引いたものです。
これは、出力側でどちら向きにどれだけずれているかを表す誤差信号です。
出力デルタが分かると、それを前の層へ順番に戻せます。

**原文参照**

PRML p.243-244 の Eq. (5.54) は、出力ユニットの `delta_k = y_k - t_k` を示す。

## Scene 5: 隠れユニットのデルタを逆向きに集める

**画面**

隠れユニット `j` から複数の出力ユニット `k` へ伸びる接続を表示する。出力側の赤いデルタが、重み `w_kj` とともに `j` へ戻る様子を示す。`h'(a_j)` を最後に掛ける。

**ナレーション**

隠れユニットは目標値と直接比べられません。そこで、そのユニットが接続している先のユニットからデルタを受け取ります。
ユニット `j` から先へ出る重みと、先のデルタを掛けて足し合わせます。
さらに、`j` 自身の活性化関数の傾き、`h'(a_j)` を掛けます。
これが、`delta_j = h'(a_j) sum_k w_kj delta_k` です。
誤差の情報は、接続を逆向きにたどりながら、各ユニットの局所微分で調整されます。

**原文参照**

PRML p.244 の Eq. (5.55)-(5.56) と Fig. 5.7 は、隠れユニットのデルタを次層のデルタから逆向きに計算する式と図を示している。

## Scene 6: 二層ネットワークで勾配を作る

**画面**

入力層、tanh 隠れ層、線形出力層の二層ネットワークを表示する。右側に、前向き式、出力デルタ、隠れデルタ、勾配式を順番に積み上げる。

**ナレーション**

具体例として、隠れ層にハイパボリックタンジェント、出力層に線形出力を持つ二層ネットワークを考えます。
前向き計算では、入力から隠れ層の `a` と `z` を作り、そこから出力 `y` を作ります。
出力デルタは、`y - t` です。隠れデルタは、`1 - z^2` を掛けて、次層の重み付きデルタを戻します。
最後に各重みの勾配は、デルタと入力側の値の外積として並びます。
全重みの勾配が、局所的な掛け算だけで一度に得られます。

**原文参照**

PRML p.245-246 の Eq. (5.58)-(5.67) は、`tanh` 隠れ層と線形出力を持つ二層ネットワークで、前向き計算、出力デルタ、隠れデルタ、重み微分をまとめている。

## Scene 7: 効率と実装確認

**画面**

左に「finite differences: `W` weights x forward pass」、右に「backprop: forward + backward」と表示し、`O(W^2)` と `O(W)` を比較する。下部に「central difference is a check, not training routine」と示す。

**ナレーション**

一回の前向き計算は、重み数を `W` とすると、およそ `O(W)` の計算です。
有限差分で全重みを一つずつ揺らすと、各重みに前向き計算が必要になり、全体は `O(W^2)` になります。
逆伝播は、一回の前向き計算と一回の後ろ向き計算で、全重みの勾配をまとめて求めるため、`O(W)` で済みます。
ただし実装確認には、少数の重みで中心差分と比べるのが有効です。
実際の学習では逆伝播で勾配を計算し、その勾配を別の最適化法が利用します。

**原文参照**

PRML p.246-247 の 5.3.3 は、有限差分では各重みごとに forward propagation が必要で `O(W^2)` になる一方、backpropagation は勾配評価を `O(W)` で行えることを説明している。Eq. (5.69) は実装確認に有効な中心差分を示している。
