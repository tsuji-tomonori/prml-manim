## Scene 1: 固定基底から学習される基底へ

**画面**

左に 3 章・4 章の固定基底関数モデル `y(x,w)=f(sum_j w_j phi_j(x))` を出す。右に `phi_j(x;theta)` と書いた学習される基底へ矢印を伸ばす。固定の曲線が、データに合わせて位置や傾きの変わる隠れユニットへ変わる様子を示す。

**ナレーション**

これまでの線形モデルでは、入力をまず固定された基底関数に通し、その出力を重みつきで足し合わせていました。
この形は扱いやすい一方で、どんな基底関数を置くかを人が先に決める必要があります。
ニューラルネットワークでは、この基底関数そのものにもパラメータを持たせます。
つまり、出力側の重みだけでなく、入力をどう特徴に変えるかも、訓練データから調整します。
この節では、その関数がどのように入力から出力まで組み立てられるかを見ます。

**原文参照**

PRML p.227 の Eq. (5.1) は固定基底関数の線形結合を示し、その後で基底関数自体をパラメータ化して学習する拡張としてニューラルネットワークを導入している。

## Scene 2: 順伝播でネットワーク関数を評価する

**画面**

入力 `x_i`、隠れユニット `z_j`、出力 `y_k` のノードを並べ、左から右へ光を流す。数式 `a_j=sum_i w_ji^(1)x_i+w_j0^(1)`、`z_j=h(a_j)`、`a_k=sum_j w_kj^(2)z_j+w_k0^(2)`、`y_k=sigma(a_k)` を順に表示する。

**ナレーション**

まず入力を重みつきで足し合わせ、隠れユニットの活性 a_j を作ります。
その活性を、シグモイドや tanh のような微分可能な非線形関数 h に通すと、隠れユニットの出力 z_j になります。
次に、z_j をもう一度重みつきで足し合わせ、出力ユニットの活性 a_k を作ります。
最後に、問題に合った出力活性化を通して y_k を得ます。
この左から右への評価が、順伝播、つまり forward propagation です。

**原文参照**

PRML p.227-228 の Eq. (5.2)-(5.7) は、入力から隠れ活性、隠れ出力、出力活性、最終出力までの変換を定義している。Fig. 5.1 は対応する 2 層ネットワーク図を示す。

## Scene 3: 隠れユニットは学習される非線形基底

**画面**

1 次元入力の上に、3 つの tanh 型隠れユニットを点線で表示する。それぞれの位置、傾き、係数を少し動かし、赤い合成曲線が target 曲線に近づく様子を示す。

**ナレーション**

隠れユニットを一つ取り出すと、入力の線形結合に非線形関数をかけた、小さな曲線として見られます。
重みとバイアスを変えると、この曲線の位置や向き、立ち上がり方が変わります。
出力層では、それらの隠れユニットをさらに重みつきで足し合わせます。
そのためネットワーク全体は、固定された基底を足すのではなく、データに合わせて動く基底を組み合わせる関数になります。
複数の隠れユニットが協調することで、一つ一つは単純でも複雑な形を作れます。

**原文参照**

PRML p.228-231 は、隠れユニットの出力を基底関数に対応づけ、Fig. 5.3 で複数の隠れユニットが関数近似に寄与する様子を示している。

## Scene 4: 出力活性化はタスクで変わる

**画面**

3 列で、回帰、複数二値分類、多クラス分類を表示する。回帰は identity、二値は logistic sigmoid、多クラスは softmax とし、出力値の範囲を図示する。

**ナレーション**

出力ユニットの活性化は、何を予測したいかで変わります。
回帰なら、出力を実数として使いたいので、恒等写像を使います。
二値分類なら、各出力を 0 から 1 の範囲に入れるため、ロジスティックシグモイドを使います。
多クラス分類なら、全クラスの確率が足して 1 になるように、softmax を使います。
この選び方は、次の 5.2 節で、尤度や誤差関数と結びついて説明されます。

**原文参照**

PRML p.228 は、標準的な回帰、複数二値分類、多クラス分類で出力活性化を使い分けることを述べている。Eq. (5.5)-(5.6) は logistic sigmoid を定義している。

## Scene 5: 非線形がなければ多層でも線形

**画面**

左に `x -> W1 -> W2 -> y` の線形だけの構成を置き、`W2 W1 x` に畳み込まれるアニメーションを出す。右に `x -> W1 -> tanh -> W2 -> y` を置き、非線形で曲線が曲がる様子を出す。

**ナレーション**

もし隠れユニットの活性化関数がすべて線形なら、何層に重ねても結果は一つの線形変換にまとめられます。
行列を何度か掛けても、全体としては別の一つの行列になるからです。
その場合、隠れ層を増やしても、本質的な表現力はあまり増えません。
一方で、途中に tanh のような非線形を入れると、線形変換だけでは作れない曲がった関数を表せます。
ニューラルネットワークの表現力は、この線形変換と非線形変換の組み合わせから生まれます。

**原文参照**

PRML p.229 は、隠れユニットが線形なら多層ネットワークは等価な隠れ層なしネットワークへ畳み込めること、連続的なシグモイド非線形がパラメータに関する微分可能性を与えることを説明している。

## Scene 6: Feed-forward topology

**画面**

閉路のない有向グラフとして一般的な feed-forward network を表示する。skip connection と sparse connection を色分けし、矢印の順序に沿ってノードの値を計算していく。

**ナレーション**

ネットワーク図は、より一般的な接続にも拡張できます。
入力から出力へ直接つなぐ skip connection を入れてもよいですし、すべての接続を持たない sparse な構造にしてもかまいません。
重要なのは、矢印をたどって閉じたサイクルがないことです。
閉路がなければ、入力が与えられたとき、前のユニットから順番に値を計算でき、出力は入力の決定的な関数になります。
この意味で、ここで扱う図は確率的グラフィカルモデルではなく、関数を表す計算グラフです。

**原文参照**

PRML p.229-230 は skip-layer connections、sparse network architecture、閉路を持たない feed-forward architecture を説明し、Eq. (5.10) で各ユニットの一般的な計算を示している。Fig. 5.2 は一般的な feed-forward topology の例である。

## Scene 7: Universal approximator の意味

**画面**

`x^2`、`sin(x)`、`|x|`、step 関数の 4 小パネルを並べ、青い target 点と赤いネットワーク近似曲線を表示する。下に「近似できること」と「学習できること」は別問題、と注記する。

**ナレーション**

十分な数の隠れユニットを使えば、フィードフォワードネットワークはとても広い範囲の関数を近似できます。
PRML では、二層ネットワークが、二乗、サイン、絶対値、ステップ関数のような形を近似する例を示しています。
この性質は、ニューラルネットワークが universal approximator と呼ばれる理由です。
ただし、近似できるパラメータが存在することと、訓練データからそのパラメータをうまく見つけられることは別です。
次の節以降では、まさにその学習方法が中心になります。

**原文参照**

## Scene 8: 重み空間の対称性

**画面**

二つの隠れユニットを持つ小さなネットワークを表示する。ひとつの隠れユニットに入る重みと出る重みを同時に符号反転しても出力が変わらないこと、隠れユニットを入れ替えても合成結果が変わらないことを示す。

**ナレーション**

最後に、同じ入出力関数を表す重みが一つとは限らない、という性質を見ておきます。
tanh は符号を反転すると出力の符号も反転します。
そこで、ある隠れユニットに入る重みを反転し、その隠れユニットから出る重みも反転すると、最終的な足し合わせは同じになります。
また、隠れユニットの順番を入れ替えても、足し合わせる項の順番が変わるだけなので、関数は変わりません。
この重み空間の対称性は、多くの場合は目立ちませんが、ベイズ的なモデル比較では後で重要になります。

**原文参照**

PRML p.231-232 の Section 5.1.1 は、tanh 隠れユニットの符号反転対称性と、隠れユニットの入れ替え対称性により、同じネットワーク関数を表す重みベクトルが複数存在することを説明している。
